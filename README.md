# 🛡️ SOC Incident Investigation & Threat Analysis

## 📌 Project Overview

This project simulates a Security Operations Center (SOC) incident investigation. It demonstrates the process of analyzing security alerts, investigating suspicious events, identifying Indicators of Compromise (IOCs), building an incident timeline, mapping attacker behavior to MITRE ATT&CK, and documenting investigation findings.

## 🎯 Project Objectives

- Analyze security alerts
- Perform alert triage
- Investigate security logs
- Identify suspicious activities
- Extract Indicators of Compromise (IOCs)
- Build an incident timeline
- Perform root cause analysis
- Map attacker behavior to MITRE ATT&CK
- Document incident findings

## 🔍 Investigation Workflow

Alert Detection → Triage → Log Analysis → IOC Identification → Timeline Analysis → MITRE ATT&CK Mapping → Root Cause Analysis → Incident Reporting

## 🚨 Incident Scenario

A suspicious security alert was identified involving unusual authentication activity and abnormal network events. The incident was investigated by correlating security logs, analyzing suspicious behavior, and identifying potential indicators of compromise.

## 🔎 Key Findings

The investigation focused on unusual login activity, suspicious network behavior, event correlation, and potential indicators of compromise.

## 🧠 MITRE ATT&CK Mapping

Attacker behavior identified during the investigation is mapped to relevant MITRE ATT&CK tactics and techniques.
